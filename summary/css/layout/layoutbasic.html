        # 设备像素和CSS像素

        • 缩放比例为100%时，CSS像素与设备像素完全重叠
        • 缩放比例缩小，CSS像素开始缩小，此时一个设备像素与几个CSS像素重叠
        • 缩放比例增大，CSS像素开始增大，此时一个CSS像素与几个设备像素重叠
        • 定义：在100％缩放级别下，一个CSS像素等于一个设备像素。

        # 屏幕尺寸和窗口尺寸

        • 屏幕尺寸：screen.width 和 screen.height 两个尺寸以设备像素为单位测量，从不改变。
        • 窗口尺寸：window.innerWidth 和 window.innerHeight 两个尺寸以CSS像素为单位。此尺寸随用户缩放而变化，放大则减小，缩小则增大。（这一点在opera浏览器上例外）测量的宽度和高度包括滚动条。它们也被视为窗口的一部分

        # 滚动偏移

        • window.pageXOffset 文档的水平滚动偏移量 以CSS像素为单位
        • window.pageYOffset 文档的垂直滚动偏移量 以CSS像素为单位
        • 从理论上讲，如果用户向上滚动然后放大，window.pageX/YOffset则会发生变化。但是，浏览器尝试通过在用户缩放时在可见页面的顶部保留相同的元素来使网页保持一致。这并不总是能完美地起作用，但这意味着实际上window.pageX/YOffset并没有真正改变：从窗口滚动出的CSS像素数量保持（大致）相同。

        # 视口

        • 桌面端浏览器只有视觉视口

        • 视口的功能是约束<html>元素：通常，所有块级元素都占用其父级宽度的100％，<body> ，<html>元素与其父元素一样宽；理论上，<html> 元素的宽度受视口宽度的限制。<html>元件采用的是视口的宽度的100％

        • 视口又恰好等于浏览器窗口：视口就是这样定义的。视口不是HTML构造，因此不能受到CSS的影响。它只是在桌面上具有浏览器窗口的宽度和高度

        • 视口的尺寸：document.documentElement.clientWidht/clientHeight css像素为单位；始终提供视口的尺寸，而不是<html> 元素的尺寸。

        • window.innerWidth和window.innerHeight也是视口的尺寸，包含了滚动条在内的尺寸，上述两个属性对不包含滚动条的尺寸，用户缩放时会发生改变。

        • <html>元素本身的尺寸：document.documentElement.offsetWidht/offsetHeight，把html当作一个块元素对待，不设置时这两个值和视口的尺寸理论上一致

        • 事件坐标：发生鼠标事件时，提供有关事件确切位置的信息

          • event.pageX/Y给出相对于<html>CSS像素中元素的坐标
          • event.clientX/Y给出相对于（视觉）视口的坐标（CSS像素为单位）
          • event.screenX\Y以设备像素为单位，给出相对于屏幕的坐标

        # 移动端

        # 视觉视口

        屏幕上显示页面的一部分。用户可以滚动以更改其看到的页面部分，或缩放以更改视觉视口的大小。以CSS像素为单位

        window.innerWidth/Height获取视觉视口的宽高，用户缩放时会发生改变。

        # 布局视口

        CSS布局采用的是布局视口，和视觉视口相比更大或者相同。以CSS像素为单位。

        布局视口的宽度和高度等于最大缩小模式下屏幕上可以显示的宽度。当用户放大时，这些尺寸保持不变。

        布局视口宽度始终相同，无论是竖屏还是横屏。

        document.documentElement.clientWidth/clientHeight获取布局视口的宽高

        布局视口有多宽？每个浏览器都有所不同。Safari iPhone使用980px，Opera 850px，Android WebKit 800px和IE 974px。

        # 屏幕与缩放等级

        • screen.width/height获取以设备像素为单位屏幕尺寸，从不改变
        • 缩放等级=screen.width / window.innerWidth

        # 媒体查询

        • 简而言之就是：您可以定义特殊的CSS规则，仅在页面宽度大于，等于或小于特定大小时才执行
        • 媒体查询的width/height针对与视口的尺寸而言，CSS像素为单位
          • media query width
        • 媒体查询的device-width/device-height针对设备尺寸而言，设备像素为单位
          • media query device-width

        # 指令

        • width。设置布局视口的宽度。
        • initial-scale。设置页面的初始缩放比例和布局视口的宽度。
        • minimum-scale。设置最小缩放级别（即用户可以缩小的程度）。
        • maximum-scale。设置最大缩放级别（即用户可以放大多少）。
        • height。应该设置布局视口的高度。任何地方都不支持它。
        • user-scalable。设置no为时，禁止用户缩放。这是一个绝对不能使用的可憎的东西。
        • device-width该width指令。它将布局视口宽度设置为理想视口宽度。

        # 理想视口

        • 给出了设备上网页的理想大小。因此，理想视口的尺寸因设备而异。
        • 两个重要的点
          • 可以将布局视口设置为理想的视口值。在width=device-width和initial-scale=1指示这样做。
          • 所有 scale指令都相对于理想视口，而不是相对于布局视口具有的任何宽度。因此maximum-scale=3，最大缩放比例是理想视口的300％。
        • 尺寸：添加<meta name="viewport" content="width=device-width,initial-scale=1">之后使用document.documentElement.clientWidth/Height可以获取理想视口的尺寸
