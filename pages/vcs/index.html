分支 ​

shell
git branch # 列举本地分支
git branch -r # 列举远端分支
git branch -a # 列举本地分支加远端分支

git cherry-pick [commit id] # 合并指定提交到当前分支
git branch -d [local branch-name] # 删除本地分支
git push origin --delete [remote branch name] # 删除远端分支

远程同步 ​

解决冲突 ​

shell
# 推送本地分支到远程仓库时，git比对文件发现远程仓库和本地分支都修改了同一个文件
# 方式一
git add .
git commit -m""
git push
# git 提示你当前分支落后了远程分支几个提交，请先取回远程分支的变化
git pull
# 发现冲突 解决冲突
git add .
git commit -m"merge: conflicts ....."
git push
# 没有冲突
git commit -m"merge branch master of ....."
git push


# 方式二
git add .
git commit -m""
git push
# git 提示你当前分支落后了远程分支几个提交，请先取回远程分支的变化
git pull --rebase
# 发现冲突 解决冲突
# merge conflicts:
#  src/ss/ss ......
git add .
git rebase --continue
# 如果有冲突会自动新建一个merge conflicts提交，自动填写了此次冲突包含的文件列表
git push

Git Flow 最佳实践 ​

Git Flow 是一种在软件开发中广泛使用的工作流程模型，它基于 Git 版本控制系统，并提供了一套清晰的分支管理策略。下面将介绍 Git Flow 的最佳实践，包括分支命名规范、分支的创建和合并、发布版本等方面。

分支命名规范 ​

在 Git Flow 中，采用一套统一的分支命名规范有助于团队成员更好地理解和管理代码。

  • master 分支：用于存储稳定的、可发布的代码。
  • develop 分支：用于集成各个功能的开发代码。
  • feature/feature-name 分支：用于开发新功能，从 develop 分支创建，开发完成后合并回 develop 分支。
  • release/version-number 分支：用于发布新版本，从 develop 分支创建，进行版本测试和准备工作，最后合并到 master 分支和 develop 分支。
  • hotfix/issue-name 分支：用于修复紧急问题，从 master 分支创建，修复完成后合并回 master 分支和 develop 分支。

示例代码：

bash
# 创建新功能分支
git checkout -b feature/user-authentication develop

# 创建新版本分支
git checkout -b release/1.0.0 develop

# 创建紧急修复分支
git checkout -b hotfix/bug-fix master

分支的创建和合并 ​

在 Git Flow 中，通过合理地创建和合并分支，可以保持代码的稳定性和可追踪性。

  • 创建新功能分支：从 develop 分支创建一个新的功能分支，进行功能开发，开发完成后合并回 develop 分支。

示例代码：

bash
git checkout -b feature/user-authentication develop
# 进行功能开发
git checkout develop
git merge --no-ff feature/user-authentication
git branch -d feature/user-authentication

  • 创建新版本分支：从 develop 分支创建一个新的版本分支，进行版本测试和准备工作，最后合并回 master 分支和 develop 分支。

示例代码：

bash
git checkout -b release/1.0.0 develop
# 进行版本测试和准备工作
git checkout master
git merge --no-ff release/1.0.0
git tag 1.0.0
git checkout develop
git merge --no-ff release/1.0.0
git branch -d release/1.0.0

发布版本 ​

在 Git Flow 中，通过合理地管理版本发布，可以实现稳定的软件交付和版本追踪。

  • 发布新版本：在 master 分支上合并一个版本分支，并打上对应的标签。

示例代码：

bash
git checkout master
git merge --no-ff release/1.0.0
git tag 1.0.0
