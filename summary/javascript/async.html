# 异步

# 异步和同步的概念

  • 异步(asynchronous) (opens new window)是指两个或两个以上的对象或事件不同时存在或发生（或多个相关事物的发生无需等待前一事物的完成）。

    • 网络与通信：异步通信是一种在双方或多方之间交换信息的方式
    • 软件设计：异步软件设计通过构建代码扩展了异步的概念，按照这种设计编写的代码使得程序能够要求一个任务与先前的一个（或多个）任务一起执行，而无需为了等待它们完成而停止执行

  • 同步(synchronous) (opens new window)是指各方都实时（或者尽可能实时）地收取（而且必要的话也处理或回复）信息的即时沟通方式。

  • 阻塞：当浏览器里的一个WEB应用进行密集运算还没有把控制权返回给浏览器的时候，整个浏览器就像冻僵了一样，这叫做阻塞；这时候浏览器无法继续处理用户的输入并执行其他任务，直到WEB应用交回处理器的控制。

  • 线程：一个线程是一个基本的处理过程，程序用它来完成任务。

  • 同步JavaScript：代码按照顺序一步一步立即执行的JavaScript代码。

# 异步JavaScript

不会按照顺序立即执行的JavaScript代码。

# 异步回调函数

将函数做为参数传递给另一个函数，然后在异步操作完成时调用该函数，以便回调可以依次对结果执行某些操作。

  • 缺点
    • 嵌套回调很麻烦且难以阅读；不优雅
    • 每层嵌套都需要故障回调，而使用promise，至于要一个catch代码块来处理整个链的错误
    • Promise回调总是按照它们放在事件队列中的严格顺序调用，异步回调不是这样
    • 当传入到一个第三方库时，异步回调对函数如何执行失去完全控制

# 合作异步JS

setTimeout和setInterval

  • setTimeout：在指定的时间后执行一段特定代码
    • 第一个参数是要运行的函数或者函数引用
    • 第二个参数是执行代码之前等待的时间间隔的数字，数字为0 函数会尽快执行(不是立即运行)
    • 更多的参数：在指定函数运行时，希望传递给函数的值
    • 清理：clearTimeout(timeout)
  • setInterval：以固定的时间间隔重复运行一段特定代码
    • 第一个参数是要运行的函数或函数引用
    • 第二个参数是时间间隔
    • 更多的参数：在指定函数运行时，希望传递给函数的值
    • 清理：clearInterval(interval)
    • 缺点：帧率未针对运行动画的系统进行优化，并且可能效率低下。除非选择特定的帧率，否则通常最好使用requestAnimationFrame()

    • 递归的setTimeout和setInterval的区别

      • 递归的setTimeout保证执行之间的延迟相同。无论代码运行多长时间，间隔都是相同的
      • setInterval选择的间隔包括执行我们想要运行的代码所花费的时间
      • 递归使用setTimeout时，每次迭代都可以在运行下一次迭代之前计算不通的延迟。第二个参数的值可以指定在再次运行代码之前等待不同的时间

    • 使用0作为setTimeout的第二个参数，函数会尽快执行，但是是在主线程代码运行之后执行

    • 你可以使用clearTimeout和clearInterval的任一一种方法来清除setTimeout和setInterval，不过为了一致性，建议一一对应的使用。

# Promise(优雅的异步处理)

允许你推迟进一步的操作，直到上一个操作完成或响应其失败。它是一个对象，代表操作的中间状态，它保证未来可能返回某结果。它能很好的解决“回调函数地狱”的问题

  • 缺点：Promise链可能很复杂，难以解析。嵌套太多，会遇到类似的麻烦的回调地狱

# async和await(让异步编程更简单)

基于promise的语法糖，使异步代码更易于编写和阅读。使用它们，异步代码看起来更像老式同步代码。

  • 缺点
    • 不能在非async函数内或代码的顶级上下文环境中使用await运算符。这有时会导致需要创建额外的函数封包。
    • 浏览器对async/await的支持不如promise那样好。
