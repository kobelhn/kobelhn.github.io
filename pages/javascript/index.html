介绍 ​

什么是JavaScript ​

JavaScript是一种广泛使用的脚本编程语言，通常用于在网页上添加交互性和动态功能。它是一种解释性语言，可以直接在网页上运行，而不需要编译。JavaScript通常用于前端开发，用于处理用户与网页的交互，例如表单验证、页面元素操作、动画效果和异步数据加载等。

JavaScript的发展历史 ​

JavaScript的发展历史可以追溯到1995年，当时由Netscape公司（现为Mozilla）的Brendan Eich开发了这门语言。起初，它被命名为LiveScript，但后来为了利用当时Java语言的热度，更名为JavaScript。然而，尽管它的名字中包含了"Java"，JavaScript与Java是两种完全不同的语言。

  • 1996年，Netscape提交了JavaScript的标准化提案给欧洲计算机制造商协会（ECMA），并且在1997年正式发布了ECMAScript 1.0标准，将JavaScript作为ECMAScript的实现之一。ECMAScript定义了JavaScript的核心语法和功能，并且成为了JavaScript的官方标准。
  • 随着时间的推移，JavaScript在Web开发中变得越来越重要。2002年，ECMAScript 4的开发开始，但由于争议和技术分歧，该标准最终被搁置。为了避免停滞不前，ECMAScript的开发团队决定采取渐进式的方式，将新功能逐步加入到标准中。
  • 2009年，ECMAScript 5（ES5）发布，引入了许多新特性，包括严格模式、JSON对象、Array增强方法等。ES5成为许多现代JavaScript代码的基础。
  • 2015年，ECMAScript 6（ES6）发布，引入了一系列重要的语言改进，例如let和const关键字、箭头函数、类、模块化等。ES6的发布标志着JavaScript的一个重要里程碑，为开发人员提供了更强大、更现代的语言特性。
  • 此后，ECMAScript开始每年发布一个新版本，每个版本都引入了一些新特性和语言改进。例如，ES7（2016年）、ES8（2017年）和ES9（2018年）分别引入了Async/Await、Object.entries()、Promise.finally()等功能。

截至（2021年9月），最新的稳定版本是ECMAScript 2021（ES12），它添加了一些新的语言特性和改进，例如逻辑赋值运算符、私有字段、Promise.any()等。此外，ECMAScript还有一些提案和实验性功能，这些功能可能在未来的版本中得到正式采纳。

JavaScript的发展历程中，除了语言本身的演进，还有许多第三方库和框架的出现，如jQuery、React、Angular和Vue.js等，它们进一步扩展了JavaScript的能力，并使得开发人员能够更高效地构建复杂的应用程序。

JavaScript版本 ​

JavaScript的版本主要是指ECMAScript（简称ES）的版本。下面是一些常见的ECMAScript版本：

  • ECMAScript 1（ES1）：这是1997年发布的第一个ECMAScript版本。
  • ECMAScript 2（ES2）：1998年发布的第二个版本，主要是对ES1的一些修正和改进。
  • ECMAScript 3（ES3）：1999年发布的版本，是一个重要的里程碑，为JavaScript带来了许多新特性，如正则表达式、try-catch语句、更严格的错误处理等。
  • ECMAScript 4（ES4）：该版本的开发在2000年代初开始，但由于争议和技术分歧，最终被放弃了。
  • ECMAScript 5（ES5）：ES5于2009年发布，引入了许多新特性，如严格模式、JSON对象、Array增强方法等。ES5成为了许多现代JavaScript代码的基础。
  • ECMAScript 6（ES6）：也被称为ES2015，于2015年发布。ES6引入了许多重要的语言改进，如let和const关键字、箭头函数、类、模块化等。ES6的发布标志着JavaScript的一个重要里程碑，为开发人员提供了更强大、更现代的语言特性。
  • ECMAScript 7（ES7）：也被称为ES2016，于2016年发布。ES7引入了一些新功能，如指数操作符、Array.includes()方法、Async/Await等。
  • ECMAScript 8（ES8）：也被称为ES2017，于2017年发布。ES8引入了一些新特性，如Object.entries()、Object.values()、异步迭代等。
  • ECMAScript 9（ES9）：也被称为ES2018，于2018年发布。ES9引入了一些新功能，如Promise.finally()、Rest/Spread属性、正则表达式命名捕获组等。
  • ECMAScript 10（ES10）：也被称为ES2019，于2019年发布。ES10引入了一些新特性，如Array.flat()、Array.flatMap()、Object.fromEntries()等。

此外，自ES6以来，ECMAScript每年都发布一个新版本，每个版本都引入了一些新特性和语言改进。较新的版本包括ES2020（ES11），ES2021（ES12）等。这些版本的发布旨在不断改进和扩展JavaScript语言的功能，以满足不断变化的开发需求。

如何运行JavaScript ​

JavaScript可以在多种环境中运行，包括浏览器、服务器和命令行。下面是一些常见的JavaScript运行方式：

  • 在浏览器中运行：JavaScript最常见的运行环境是浏览器。可以通过在HTML文件中使用<script>标签来嵌入JavaScript代码，并在浏览器中打开HTML文件来执行JavaScript代码。可以将JavaScript代码直接写在<script>标签中，也可以将JavaScript代码写在外部文件中，然后使用src属性引入外部文件。

  • 在服务器端运行：JavaScript也可以在服务器端运行，常用的服务器端JavaScript运行环境是Node.js。首先，需要安装Node.js环境。然后，可以使用文本编辑器创建一个.js文件，并在文件中编写JavaScript代码。保存文件后，在命令行中使用node命令来运行JavaScript文件。

  • 在命令行中运行：除了Node.js之外，还可以在命令行中直接运行一些简单的JavaScript代码。在命令行中输入node命令，进入Node.js的交互式模式，然后可以直接输入JavaScript代码并按回车执行。

除了上述方式，JavaScript还可以在其他环境中运行，如桌面应用程序（使用Electron）、移动应用程序（使用React Native）等。选择适合你需求的环境，并按照相应的方式来运行JavaScript代码。

变量命名规则 ​

  • 变量名只能包含字母、数字、美元符号（$）和下划线（_），不能包含空格或其他标点符号。
  • 变量名不能以数字开头。
  • 变量名不能使用JavaScript关键字和保留字。
  • 变量名区分大小写。
  • 变量名应该使用驼峰命名法（camelCase）。
  • 变量名应该具有描述性。
  • 变量名应该尽可能简短，但不要太过简单。
  • 变量名应该遵循一致的命名规则。
  • 变量名应该避免使用缩写。

对象常用方法 ​

  1. Object.keys(obj)：返回一个包含对象自身可枚举属性名称的数组。
  2. Object.values(obj)：返回一个包含对象自身可枚举属性值的数组。
  3. Object.entries(obj)：返回一个包含对象自身可枚举属性键值对的数组。
  4. Object.assign(target, source1, source2, ...)：将一个或多个源对象的属性复制到目标对象，并返回目标对象。
  5. Object.create(proto, [propertiesObject])：使用指定的原型对象创建一个新对象。
  6. Object.defineProperty(obj, prop, descriptor)：定义对象的新属性或修改现有属性的特性。
  7. Object.getOwnPropertyDescriptor(obj, prop)：获取对象属性的特性描述符。
  8. Object.getOwnPropertyNames(obj)：返回一个包含对象自身所有属性名称（包括不可枚举属性）的数组。
  9. Object.getPrototypeOf(obj)：获取对象的原型。
  10. Object.setPrototypeOf(obj, proto)：设置对象的原型。
  11. Object.hasOwnProperty(prop)：检查对象是否具有指定名称的自身属性。
  12. Object.is(obj1, obj2)：比较两个值是否严格相等。
  13. Object.toString()：返回对象的字符串表示。

原型链与继承 ​

对象原型是JavaScript中的一种机制，它定义了对象如何共享属性和方法。每个对象在创建时都与一个原型相关联，通过该原型，对象可以访问并继承其属性和方法。

在JavaScript中，对象可以通过字面量、构造函数或Object.create()等方式创建。每个对象都有一个内部属性[[Prototype]]，它引用了该对象的原型。当我们尝试访问对象的属性或方法时，如果对象本身没有找到，JavaScript会沿着原型链向上查找，直到找到对应的属性或方法，或者到达原型链的顶端（通常是Object.prototype）。

通过原型链，对象可以继承来自其原型的属性和方法，这使得在JavaScript中实现对象的属性和方法共享成为可能。当我们在对象上调用一个方法时，JavaScript引擎会先检查对象自身是否具有该方法，如果没有，则会继续检查原型链上的原型对象，直到找到该方法为止。

原型链的最顶端是Object.prototype，它是所有对象的默认原型。它包含了一些常用的方法，比如toString()和hasOwnProperty()。我们也可以通过将对象的原型设置为其他对象来实现对象之间的继承关系，这样子对象就可以共享父对象原型上的属性和方法。

Error ​

当代码运行时发生错误，会创建新的Error对象并将其抛出。

错误类型:

  • EvalError: 错误原因与eval()有关。
  • RangeError: 错误原因是数值变量或参数超出其有效范围。
  • ReferenceError: 错误原因是非法或不能识别的引用数值。
  • SyntaxError: 错误原因是语法错误。
  • TypeError: 错误原因是变量或参数不属于有效类型。
  • URIError: 错误原因是encodeURI()或decodeURI()的参数无效。
  • AggregateError: 用于表示多个错误的错误。
  • InternalError: 创建了JavaScript引擎无法执行的错误。

构造函数:

  • new Error([message[, fileName[, lineNumber]]]) || new Error([message[, options]])
    • message: 可选。人类可读的错误描述。
    • fileName: 可选。创建错误的代码文件名。
    • lineNumber: 可选。创建错误的代码行号。
    • options: 可选。一个对象，包含以下属性：
      • cause: 可选。一个错误或字符串，表示导致此错误的原因。
  • 只有在 options 参数为对象时才会尝试读取 options.cause 属性，这样可以避免与另一种非标准的 Error(message, fileName, lineNumber) 函数签名产生歧义，后者要求第二个参数必须是字符串。如果你省略了 options 参数，或者将原始值作为 options 传入，又或者传递的对象中没有 cause 属性，那么创建的 Error 对象将不会包含 cause 属性。
javascript
const x = new Error("error message");

try {
  frameworkThatCanThrow();
} catch (err) {
  throw new Error("New error message", { cause: err });
}

常用静态方法 ​
  • Array.from()：从类似数组或可迭代对象创建一个新的数组实例。
    • 结合Set可以快速对数组进行去重: Array.from(new Set(arr))
    • 在DOM操作中，可以将NodeList转换为数组: Array.from(document.querySelectorAll('div'))，方便使用数组的方法。
    • Array.from()还可以接受第二个参数，类似于数组的map()方法，用来对每个元素进行处理，将处理后的值放入返回的数组。
  • Array.isArray()：判断一个值是否为数组。
常用的原型方法 ​

Array.prototype.*, * 代表方法名

  • concat(): 用于合并两个或多个数组。此方法不会更改现有数组，而是返回一个新数组。
    • arr.concat(value1, value2, ..., valueN)
    • 返回值：一个新的数组实例。
    • 是否会影响原数组：不会。
  • every(): 测试数组的所有元素是否都通过了指定函数的测试。
    • arr.every(callback(element[, index[, array]])[, thisArg])
    • 返回值：如果回调函数的每一次返回都为 truthy 值，返回 true ，否则返回 false 。
    • 是否会影响原数组：不会。
  • some(): 测试数组中的某些元素是否通过由提供的函数实现的测试。
    • arr.some(callback(element[, index[, array]])[, thisArg])
    • 返回值：如果回调函数的任一次返回为 truthy 值，返回 true ，否则返回 false 。
    • 是否会影响原数组：不会。
  • filter(): 创建一个新数组, 其包含通过所提供函数实现的测试的所有元素。
    • arr.filter(callback(element[, index[, array]])[, thisArg])
    • 返回值：一个新的、由通过测试的元素组成的数组，如果没有任何数组元素通过测试，则返回空数组。
    • 是否会影响原数组：不会。
  • sort(): 对数组的元素进行排序，并返回数组。默认排序顺序是根据字符串Unicode码点。
    • arr.sort([compareFunction])
    • 返回值：排序后的数组。请注意，数组已原地排序，并且不进行复制。
    • 是否会影响原数组：会。
  • slice(): 返回一个新的数组对象，这一对象是一个由 begin 和 end 决定的原数组的浅拷贝（包括 begin，不包括 end）。
    • arr.slice([begin[, end]])
    • 返回值：一个含有提取元素的新数组。
    • 是否会影响原数组：不会。
    • 使用此方法可快速复制一个数组: var arr2 = arr.slice(0);, arr2 是 arr 的浅拷贝。
  • splice(): 通过删除现有元素和/或添加新元素来更改一个数组的内容。
    • array.splice(start[, deleteCount[, item1[, item2[, ...]]]])
    • 返回值：由被删除的元素组成的一个数组。如果只删除了一个元素，则返回只包含一个元素的数组。如果没有删除元素，则返回空数组。
    • 是否会影响原数组：会。
  • forEach(): 对数组的每个元素执行一次提供的函数。
    • arr.forEach(callback(currentValue[, index[, array]])[, thisArg])
    • 返回值：undefined。
    • 是否会影响原数组：不会。
  • map(): 创建一个新数组，其结果是该数组中的每个元素是调用一次提供的函数后的返回值。
    • arr.map(callback(currentValue[, index[, array]])[, thisArg])
    • 返回值：一个新数组，每个元素都是回调函数的结果。
    • 是否会影响原数组：不会。
  • reduce(): 对累加器和数组中的每个元素（从左到右）应用一个函数，将其减少为单个值。
    • arr.reduce(callback(accumulator, currentValue[, index[, array]])[, initialValue])
    • 返回值：函数累计处理的结果。
    • 是否会影响原数组：不会。
  • pop(): 从数组中删除最后一个元素，并返回该元素的值。此方法更改数组的长度。
    • arr.pop()
    • 返回值：从数组中删除的元素。
    • 是否会影响原数组：会。
  • push(): 将一个或多个元素添加到数组的末尾，并返回该数组的新长度（该方法修改原有数组）。
    • arr.push(element1[, ...[, elementN]])
    • 返回值：数组的新长度。
    • 是否会影响原数组：会。
  • shift(): 从数组中删除第一个元素，并返回该元素的值。此方法更改数组的长度。
    • arr.shift()
    • 返回值：从数组中删除的元素。
    • 是否会影响原数组：会。
  • unshift(): 将一个或多个元素添加到数组的开头，并返回该数组的新长度（该方法修改原有数组）。
    • arr.unshift(element1[, ...[, elementN]])
    • 返回值：数组的新长度。
    • 是否会影响原数组：会。
  • find(): 返回数组中满足提供的测试函数的第一个元素的值。否则返回 undefined。
    • arr.find(callback(element[, index[, array]])[, thisArg])
    • 返回值：数组中满足提供的测试函数的第一个元素的值。否则返回 undefined。
    • 是否会影响原数组：不会。
  • findIndex(): 返回数组中满足提供的测试函数的第一个元素的索引。否则返回-1。
    • arr.findIndex(callback(element[, index[, array]])[, thisArg])
    • 返回值：数组中满足提供的测试函数的第一个元素的索引。否则返回-1。
    • 是否会影响原数组：不会。
  • indexOf(): 返回在数组中可以找到一个给定元素的第一个索引，如果不存在，则返回-1。
    • arr.indexOf(searchElement[, fromIndex])
    • 返回值：在数组中找到的元素的第一个索引，如果不存在，则返回-1。
    • 是否会影响原数组：不会。
  • lastIndexOf(): 返回指定元素（也即有效的 JavaScript 值或变量）在数组中的最后一个的索引，如果不存在则返回 -1。从数组的后面向前查找，从 fromIndex 处开始。
    • arr.lastIndexOf(searchElement[, fromIndex])
    • 返回值：指定元素在数组中的最后一个的索引，如果不存在则返回 -1。
    • 是否会影响原数组：不会。
  • includes(): 判断一个数组是否包含一个指定的值，根据情况，如果包含则返回 true，否则返回false。
    • arr.includes(valueToFind[, fromIndex])
    • 返回值：如果数组包含指定的值，则为 true，否则为 false。
    • 是否会影响原数组：不会。
  • join(): 将一个数组（或一个类数组对象）的所有元素连接成一个字符串并返回这个字符串。
    • arr.join([separator])
    • 返回值：一个所有数组元素连接的字符串。如果 arr.length 为0，则返回空字符串。
    • 是否会影响原数组：不会。
  • reverse(): 将数组中元素的位置颠倒，并返回该数组。该方法会改变原数组。
    • arr.reverse()
    • 返回值：颠倒后的数组。
    • 是否会影响原数组：会。
  • flat(): 创建一个新数组，其中包括将数组与所有子数组（递归）连接在一起的指定深度的新数组。
    • var newArray = arr.flat([depth])
    • 返回值：一个包含将数组与所有子数组连接在一起的新数组。
    • 是否会影响原数组：不会。
javascript
// reduce

const arr = [1, 2, 3, 4, 5];
const sum = arr.reduce((acc, cur) => acc + cur, 0);
console.log(sum); // 15

// flat

const arr = [1, 2, [3, 4, [5, 6]]];
const flatArr = arr.flat(2);
console.log(flatArr); // [1, 2, 3, 4, 5, 6]
console.log(arr); // [1, 2, [3, 4, [5, 6]]]

相等比较 ​

更多细节「点击这里」

=== ​

===和!==

  • 如果操作数的类型不同，则返回 false。
  • 如果两个操作数都是对象，只有当它们指向同一个对象时才返回 true。
  • 如果两个操作数都为 null，或者两个操作数都为 undefined，返回 true。
  • 如果两个操作数有任意一个为 NaN，返回 false。
  • 否则，比较两个操作数的值：
    • 数字类型必须拥有相同的数值。+0 和 -0 会被认为是相同的值。
    • 字符串类型必须拥有相同顺序的相同字符。
    • 布尔运算符必须同时为 true 或同时为 false。

== ​

==和!=

  • 如果操作数具有相同的类型，则按如下方式进行比较：
    • 对象（Object）：仅当两个操作数引用同一个对象时返回 true。
    • 字符串（String）：只有当两个操作数具有相同的字符且顺序相同时才返回 true。
    • 数值（Number）：如果两个操作数的值相同，则返回 true。+0 和 -0 被视为相同的值。如果任何一个操作数是 NaN，返回 false；所以，NaN 永远不等于 NaN。
    • 布尔值（Boolean）：仅当操作数都为 true 或都为 false 时返回 true。
    • 大整形（BigInt）：仅当两个操作数值相同时返回 true。
    • 符号（Symbol）：仅当两个操作数引用相同的符号时返回 true。
  • 如果其中一个操作数为 null 或 undefined，另一个操作数也必须为 null 或 undefined 以返回 true。否则返回 false。
  • 如果其中一个操作数是对象，另一个是基本类型，按此顺序使用对象的 @@toPrimitive()（以 "default" 作为提示），valueOf() 和 toString() 方法将对象转换为基本类型。（这个基本类型转换与相加中使用的转换相同。）
  • 在这一步，两个操作数都被转换为基本类型（String、Number、Boolean、Symbol 和 BigInt 中的一个）。其余的转换是逐个进行的。
    • 如果是相同的类型，使用步骤 1 进行比较。
    • 如果其中一个操作数是 Symbol 而另一个不是，返回 false。
    • 如果其中一个操作数是布尔型而另一个不是，则将布尔型转换为数字：true 转换为 1，false 转换为 0。然后再次松散地比较两个操作数。
    • Number to String：使用与 Number() 构造函数相同的算法将字符串转换为数字。转换失败将导致 NaN，这将保证相等是 false。
    • Number to BigInt：按数值进行比较。如果数值为 ±∞ 或 NaN，返回 false。
    • String to BigInt：使用与 BigInt() 构造函数相同的算法将字符串转换为 BigInt。如果转换失败，返回 false。

Object.is() ​

  • 接收两个参数，返回值是一个确定的布尔值
  • 静态方法确定两个值是否为相同值，以下任意一项成立时则两个值相同
    • 都是 undefined
    • 都是 null
    • 都是 true 或者都是 false
    • 都是长度相同、字符相同、顺序相同的字符串
    • 都是相同的对象（意味着两个值都引用了内存中的同一对象）
    • 都是 BigInt 且具有相同的数值
    • 都是 symbol 且引用相同的 symbol 值
    • 都是数字且
    • 都是 +0
    • 都是 -0
    • 都是 NaN
    • 都有相同的值，非零且都不是 NaN

for语句 ​

一个for循环会一直重复执行，直到指定的循坏条件为false。

javascript

for ([initialExpression]; [conditionExpression]; [incrementExpression])
  statement or { statements }
1
2
3

执行步骤如下:

  1. 如果有初始化表达式 initialExpression，它将被执行。这个表达式通常会初始化一个或多个循环计数器，但语法上是允许一个任意复杂度的表达式的。这个表达式也可以声明变量。
  2. 计算 condition 表达式的值。如果 condition 的值是 true，循环中的语句会被执行。如果 condition 的值是 false，for 循环终止。如果 condition 表达式整个都被省略掉了，condition 的值会被认为是 true。
  3. 循环中的 statement 被执行。如果需要执行多条语句，可以使用块（{ ... }）来包裹这些语句。
  4. 如果有更新表达式 incrementExpression，执行更新表达式。
  5. 回到步骤 2。

do...while语句 ​

javascript

do
  statement or { statements }
while (condition);

3
4

执行步骤如下:

  1. statement 在检查条件之前会执行一次。要执行多条语句（语句块），要使用块语句（{ ... }）包括起来。
  2. 如果 condition 为真（true），statement 将再次执行。在每个执行的结尾会进行条件的检查。
  3. 当 condition 为假（false），执行会停止并且把控制权交回给 do...while 后面的语句。

while语句 ​

javascript

while (condition)
  statement or { statements }
1
2
3

执行步骤如下:

  1. 只要指定的条件求值为真（true）就会一直执行它的语句块。
  2. 如果这个条件变为假，循环里的 statement 将会停止执行并把控制权交回给 while 语句后面的代码。
  3. 条件检测会在每次 statement 执行之前发生。如果条件返回为真， statement 会被执行并紧接着再次测试条件。如果条件返回为假，执行将停止并把控制权交回给 while 后面的语句。

IIFE ​

更多细节「点击这里」

立即调用函数表达式是一个定义时就会立即执行的JavaScript函数，它是一种设计模式，也被称为自执行匿名函数。

  • 第一部分是一个具有词法作用域的匿名函数，并且用圆括号运算符 () 运算符闭合起来。这样不但阻止了外界访问 IIFE 中的变量，而且不会污染全局作用域。
  • 第二部分创建了一个立即执行函数表达式 ()，通过它，JavaScript 引擎将立即执行该函数。

  • 避免污染全局命名空间、避免命名冲突

(() => {
  // 初始化代码
  let firstVariable;
  let secondVariable;
})();

(() => {
  // 初始化代码
  let firstVariable;
  let secondVariable;
})();

// firstVariable 和 secondVariable 变量在函数执行后会被丢弃

arguments对象 ​

函数的实际参数会被保存在一个类数组的arguments对象中，在函数内可以通过arguments[i]来访问实际参数。

javascript
function myConcat(separator) {
  let result = ""; // 初始化列表
  // 迭代 arguments
  for (let i = 1; i < arguments.length; i++) {
    result += arguments[i] + separator;
  }
  return result;
}

函数作用域 ​

在函数内定义的变量不能在函数之外的任何地方访问，因为变量仅仅在该函数的作用域内定义。相对应的，一个函数可以访问定义在其范围内的任何变量和函数。定义在全局域中的函数可以访问所有定义在全局域中的变量。

关于作用域的细节不多赘述，可以参考「JavaScript-变量-变量作用域」进行了解。

闭包 ​

这里推荐《你不知道的JavaScript》系列书籍，关于JavaScript的基础，这本书是个人觉得最好的，可以帮助你更好的理解JavaScript的基础知识。 该书中对闭包的定义是: 当函数可以记住并访问所在的词法作用域时，就产生了闭包，即使函数是在当前词法作用域之外执行 关于词法作用域建议阅读该书以获取相关知识，书籍资源可联系邮箱获取。

个人的一句话理解就是: 函数内部定义一个内部函数，函数在外部被调用时这个内部函数依然可以访问函数内的变量，这就形成了一个闭包。

闭包无处不在，只要使用了函数，就使用了闭包。闭包的使用场景也很多，比如：模块模式、函数柯里化、函数记忆等等。关于说闭包的缺点，会造成内存泄漏，这个问题可以交给垃圾回收机制来处理，可以放心大胆的使用闭包。

事件循环机制（Event Loop） ​

JavaScript中的事件循环（Event Loop）是一种用于处理异步操作的机制。它是JavaScript单线程执行模型的核心组成部分，用于处理事件、回调函数和异步任务。

事件循环的原理是基于一个事件队列（Event Queue）和一个执行栈（Execution Stack）。

首先，JavaScript引擎会执行所有的同步代码，并将异步任务（如定时器、网络请求、事件监听等）交给相应的API处理。当异步任务完成时，它们会被添加到事件队列中。

事件队列是一个先进先出（FIFO）的数据结构，保存着待执行的事件和回调函数。当主线程空闲时，事件循环机制会从事件队列中取出一个事件，并将其对应的回调函数放入执行栈中执行。

执行栈是一个存储函数调用的栈结构，每当一个函数被调用时，它就会被添加到执行栈的顶部。当函数执行完毕，它会从执行栈中移除。

事件循环的关键是不断地从事件队列中取出事件并执行对应的回调函数。这个过程会不断重复，形成一个循环。因此，这个机制被称为事件循环。

需要注意的是，JavaScript是单线程执行的，意味着一次只能执行一个任务。当事件循环执行一个任务时，它会一直执行，直到该任务执行完毕或者遇到需要等待的异步操作。这也是为什么长时间运行的JavaScript代码会导致页面失去响应的原因。

回调函数 ​

  • 回调函数是一个被传递到另一个函数中的会在适当的时候被调用的函数。
  • 回调函数可以是同步的或者异步的。
  • 回调函数可以是命名函数或者匿名函数。
  • 注意：回调函数不是由它所在的函数的调用者提供的，而是由其所在的函数来定义的。
  • 回调函数的使用场景很多，比如：事件处理函数、异步请求、定时器等等。
  • 注意避免回调地狱
    • 回调地狱指的是多个回调函数嵌套调用，导致代码可读性差，难以维护。
    • 解决方案：使用Promise、async/await等

Promise ​

关于Promise可以参考阮一峰的ES6入门教程进行学习。在平常编程中Promise的应用十分广泛，这里只列举一下其常用的静态方法和原型方法。细节请自主学习

  • Promise.all(): 接收一个Promise对象的数组作为参数，当这个数组里的所有Promise对象全部变为resolve或reject状态的时候，它才会去调用then方法。
  • Promise.resolve(): 将现有对象转为Promise对象。
  • Promise.reject(): 返回一个新的Promise实例，该实例的状态为rejected。
  • Promise.prototype.then(): 为Promise实例添加状态改变时的回调函数。
  • Promise.prototype.catch(): 用于指定发生错误时的回调函数。
  • Promise.prototype.finally(): 用于指定不管 Promise 对象最后状态如何，都会执行的操作。

setTimeOut ​

  • setTimeout() 方法用于在指定的毫秒数后调用函数或计算表达式。
  • setTimeout() 只执行一次指定函数。如果要多次调用，请使用 setInterval() 或者让函数自身再次调用 setTimeout()。
  • setTimeout() 函数返回值是一个整数，表示定时器的编号，这个值是计划执行代码的唯一标识符。这个值可以传递给 clearTimeout() 函数来取消该定时器。const timer = setTimeout(() => {}, 1000); clearTimeout(timer);

setInterval ​

  • setInterval() 方法重复调用一个函数或执行一个代码段，在每次调用之间具有固定的时间延迟。
  • clearInterval() 方法用于停止 setInterval() 方法执行的函数代码。
  • setInterval() 方法会不停地调用函数，直到 clearInterval() 被调用或窗口被关闭。
  • 由 setInterval() 返回的 ID 值可用作 clearInterval() 方法的参数。const timer = setInterval(() => {}, 1000); clearInterval(timer);

垃圾回收 ​

  • 引用：在内存管理的环境中，一个对象如果有访问另一个对象的权限（隐式或者显式），叫做一个对象引用另一个对象。
  • 引用计数垃圾收集：这是最初级的垃圾收集算法。此算法把“对象是否不再需要”简化定义为“对象有没有其他对象引用到它”。如果没有引用指向该对象（零引用），对象将被垃圾回收机制回收。
    • 循环引用：无法处理循环引用的事例。
  • 标记-清除算法：这个算法把“对象是否不再需要”简化定义为“对象是否可以获得”。
    • 这个算法假定设置一个叫做根（root）的对象（在 Javascript 里，根是全局对象）。垃圾回收器将定期从根开始，找所有从根开始引用的对象，然后找这些对象引用的对象……从根开始，垃圾回收器将找到所有可以获得的对象和收集所有不能获得的对象。
