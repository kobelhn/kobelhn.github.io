# 值类型（基本类型）⇧

JavaScript会自动将基本类型转换为基本类型对象，只有将基本类型转化为基本类型对象之后才可以使用基本类型对象的方法。当基本类型需要调用一个基本类型对象才有的方法或者查询值的时候(基本类型值是没有这些方法的)，JavaScript 会自动将基本类型转化为基本类型对象并且调用相应的方法或者执行查询。

# JavaScript中对值类型和引用类型的理解以及它们的区别

参考链接：https://www.cnblogs.com/leiting/p/8081413.html

  • 1）值类型

    • 占用空间固定，保存在栈中

      当一个方法执行时，每个方法都会建立自己的内存栈，在这个方法内定义的变量将会逐个放入这块栈内存中，随着方法的执行结束，这个方法的栈内存也将自然销毁了。因此，所有在方法中定义的变量都是放在栈内存中；栈中存储的是基础变量以及一些对象的引用变量，基础变量的值是存储在栈中，而引用变量存储在栈中的是指向堆中的数组或者对象的地址，这就是为何修改引用类型总会影响到其他指向这个地址的引用变量。

    • 保存与复制的是值本身

    • 使用typeof检测数据类型

  • 引用类型

    • 占用空间不固定，保存在堆中

      当我们在程序中创建一个对象时，这个对象将被保存到运行时数据区中，以便反复利用。这个运行时数据区就是堆内存。堆内存中的对象不会随方法的结束而销毁，即使方法结束后，这个对象还可能被另一个引用变量所引用，则这个对象依然不会被销毁，只有当一个对象没有任何引用变量引用它时，系统的垃圾回收机制才会在核实的时候回收它。

    • 保存与复制的是对象的一个指针

    • 使用instanceof检测数据类型

# 作用域⇧

  • 作用域是负责收集并维护由所有声明的标识符(变量)组成的一系列查询，并实施一套非常严格的规则，确定当前执行的代码对这些标识符(变量)的访问权限

  • 作用域嵌套：当一个块或函数嵌套在另一个块或函数中时，就会产生作用域嵌套。

  • 作用域查找：作用域查找会在找到第一个匹配的标识符时停止；引擎从当前的执行作用域开始查找变量，如果找不到就去上一级继续查找，当抵达最外层的作用域时，无论是否找到，查找过程都会停止。

  • 作用域规则出现的理由：编程语言最基本的功能之一是能够储存变量的值，并能在之后对这个值进行访问与修改，这种存储与访问变量的能力给程序增加了状态。失去状态，程序会受到高度限制，做不到有趣。

  • 词法作用域：定义在词法阶段的作用域，由书写代码时将变量和块作用域写在哪里决定的。

    • 作用域气泡由其对应的作用域代码写在哪里决定，逐级包含。
    • eval 和 with 会欺骗词法作用域，引擎在发现了这两种情况时，关于标识符的位置判断就会失效，会影响性能

  • 函数作用域：属于这个函数的全部变量都可以在整个函数的范围内使用和复用

    • 利用函数作用域可以隐藏内部实现

      • 最小授权或最小暴露原则
      • 避免同名标识符之间的命名冲突
        • 隔离全局命名空间
        • 模块化管理

    • 匿名函数表达式：始终给函数表达式命名是一个最佳实践

    • 立即执行函数表达式(IIFE)

      • 当成函数调用并传递参数。如：(function() {})(window)

      • 倒置代码的运行顺序，把需要运行的函数放到第二位，IIFE执行之后当作参数传递进去

        (function IIFE(def) {def(window)})(function def(global) { .... })

  • 块级作用域：表面上看JavaScript并没有块级作用域的相关功能。实际可以通过let/const 或者 try/catch 可以创建一个块级作用域

  • 提升：变量和函数声明会从它们在代码中出现的位置被“移动”到了最上面。这个过程叫做提升。只有声明会被提升，而赋值或其他运行逻辑会留在原地。函数声明会优先被提升，然后是变量声明。

# 闭包⇧

  • 定义：当函数可以记住并访问所在的词法作用域时，就产生了闭包，即使函数是在当前词法作用域之外执行的。
  • 闭包最大的应用是模块
    • 模块模式的两个必要条件
      • 必须有外部的封闭函数，该函数至少被调用一次。
      • 封闭函数必须返回至少一个内部函数，这样内部函数才能在私有作用域中形成闭包，并且可以访问或者修改私有的状态。

# 对象原型/原型链⇧

  • __proto__：JavaScript中的对象都有一个特殊的__proto__内置属性，其实就是对于其他对象的引用。几乎所有的对象在创建时__proto__属性都会被赋予一个非空的值。所有普通的__proto__链最终都会指向内值的Object.prototype。
  • 引用对象的属性第一步是检查对象本身是否有这个属性，如果有就使用它，如果无法在对象本身找到需要的属性就会继续访问对象的原型链，这个过程会持续到找到匹配的属性或者查找完整的原型链。通过各种语法进行属性查找时都会查找原型链，直到找到属性或者查找完整原型链
  • 属性屏蔽（其他语言里的重写）例：obj.a = a
    • 如果原型链上层存在同名属性，如果这个同名属性只是普通数据访问属性，没有被标记为只读，那就会在当前对象中添加一个新属性
    • 如果原型链上层存在同名属性，这个同名属性又被设置为只读，那这个属性不会被添加，也不会改变上层的同名属性，这条语句相当于被忽略，严格模式下程序会报错。
    • 如果原型链上层存在同名属性，这个同名属性是一个setter，那么会执行setter，属性不会添加到当前对象，也不会改变setter
  • 继承
    • 原型继承：当前对象可以访问和引用它原型链上的属性
    • 构造函数继承：所有带new关键字的函数调用就是构造函数
  • 性能：
    • 在原型链上查找属性比较耗时，对性能有副作用，这在性能要求苛刻的情况下很重要。在试图访问不存在的属性时会遍历整个原型链
    • hasOwnProperty()是JavaScript中唯一一个处理属性并且不会遍历原型链的方法。另一种这样的方法是：Object.keys() 只会遍历当前对象的属性而不会遍历原型链。

# this全面解析⇧

  • this是在运行时绑定的，并不是在编写时绑定，它的上下文取决于函数调用时的各种条件。this的绑定和函数声明的位置没有任何关系，只取决于函数的调用方式。函数调用时，会创建一个执行上下文，这个上下文会包含函数在哪里被调用，函数的调用方法，函数的参数等信息，this就是执行上下文中的一个属性，会在函数执行的过程中用到。

  • 绑定方式：

    • 默认绑定：独立函数调用。严格模式下，全局对象无法使用默认绑定，程序会抛错误。

    • 隐式绑定：调用位置是否有上下文对象，或者说是否被某个对象拥有或者包含。

      function foo() { console.log(this.a) }; const obj = { foo, a: 2 }; obj.foo(); // 2

    • 显式绑定：call() apply()方法的参数，第一个参数是一个对象，会把这个对象绑定到this，在调用时指定这个this

      • 硬绑定：使用Function.prototype.bind()方法，直接绑定函数调用时的this

    • new绑定：使用new来调用函数（构造函数调用）时

      • 创建（构造）一个全新的对象
      • 这个新对象会被执行原型连接
      • 这个对象会被绑定到函数调用的this
      • 如果函数没有返回其他对象，那么new表达式中的函数调用，会自动返回这个新对象

  • 绑定优先级（优先级依次降低）

    • new绑定
    • 显式绑定，硬绑定
    • 隐式绑定
    • 默认绑定

  • 绑定例外：

    • 如果把null或者undefined作为this的绑定对象传入call,apply或bind中，这些值在调用时会被忽略而应用默认绑定规则

      function foo() { console.log(this.a) }; var a = 2; foo.call(null); // 2

    • 有可能会在有意无意间创建一个函数的间接引用，这时也会应用默认绑定规则

# 异步和同步的概念⇧

  • 异步(asynchronous) (opens new window)是指两个或两个以上的对象或事件不同时存在或发生（或多个相关事物的发生无需等待前一事物的完成）。

    • 网络与通信：异步通信是一种在双方或多方之间交换信息的方式
    • 软件设计：异步软件设计通过构建代码扩展了异步的概念，按照这种设计编写的代码使得程序能够要求一个任务与先前的一个（或多个）任务一起执行，而无需为了等待它们完成而停止执行

  • 同步(synchronous) (opens new window)是指各方都实时（或者尽可能实时）地收取（而且必要的话也处理或回复）信息的即时沟通方式。

  • 阻塞：当浏览器里的一个WEB应用进行密集运算还没有把控制权返回给浏览器的时候，整个浏览器就像冻僵了一样，这叫做阻塞；这时候浏览器无法继续处理用户的输入并执行其他任务，直到WEB应用交回处理器的控制。

  • 线程：一个线程是一个基本的处理过程，程序用它来完成任务。

  • 同步JavaScript：代码按照顺序一步一步立即执行的JavaScript代码。

# 异步回调函数

将函数做为参数传递给另一个函数，然后在异步操作完成时调用该函数，以便回调可以依次对结果执行某些操作。

  • 缺点
    • 嵌套回调很麻烦且难以阅读；不优雅
    • 每层嵌套都需要故障回调，而使用promise，至于要一个catch代码块来处理整个链的错误
    • Promise回调总是按照它们放在事件队列中的严格顺序调用，异步回调不是这样
    • 当传入到一个第三方库时，异步回调对函数如何执行失去完全控制

# 合作异步JS

setTimeout和setInterval

  • setTimeout：在指定的时间后执行一段特定代码
    • 第一个参数是要运行的函数或者函数引用
    • 第二个参数是执行代码之前等待的时间间隔的数字，数字为0 函数会尽快执行(不是立即运行)
    • 更多的参数：在指定函数运行时，希望传递给函数的值
    • 清理：clearTimeout(timeout)
  • setInterval：以固定的时间间隔重复运行一段特定代码
    • 第一个参数是要运行的函数或函数引用
    • 第二个参数是时间间隔
    • 更多的参数：在指定函数运行时，希望传递给函数的值
    • 清理：clearInterval(interval)
    • 缺点：帧率未针对运行动画的系统进行优化，并且可能效率低下。除非选择特定的帧率，否则通常最好使用requestAnimationFrame()
  • 需要注意的点

    • 递归的setTimeout和setInterval的区别

      let i = 1;

setTimeout(function run() {
  console.log(i);
  i++;
  setTimeout(run, 100);
}, 100);

      let i = 1;

setInterval(function run() {
  console.log(i);
  i++
}, 100);
      • 递归的setTimeout保证执行之间的延迟相同。无论代码运行多长时间，间隔都是相同的
      • setInterval选择的间隔包括执行我们想要运行的代码所花费的时间
      • 递归使用setTimeout时，每次迭代都可以在运行下一次迭代之前计算不通的延迟。第二个参数的值可以指定在再次运行代码之前等待不同的时间

    • 使用0作为setTimeout的第二个参数，函数会尽快执行，但是是在主线程代码运行之后执行

    • 你可以使用clearTimeout和clearInterval的任一一种方法来清除setTimeout和setInterval，不过为了一致性，建议一一对应的使用。

# Promise(优雅的异步处理)

允许你推迟进一步的操作，直到上一个操作完成或响应其失败。它是一个对象，代表操作的中间状态，它保证未来可能返回某结果。它能很好的解决“回调函数地狱”的问题

  • 缺点：Promise链可能很复杂，难以解析。嵌套太多，会遇到类似的麻烦的回调地狱

# 并发模型以及事件循环⇧

JavaScript有一个基于事件循环的并发模型，事件循环负责执行代码、收集和处理事件以及执行队列中的子任务。

# JavaScript运行时⇧

  • 栈(Stack)：函数调用形成了一个由若干帧组成的栈
  • 堆(Heap)：对象被分配在堆中，堆是一个用来表示一大块(通常是非结构化的)内存区域的计算机术语
  • 队列(Queue)：一个JavaScript运行时包含了一个待处理消息的消息队列。

# 事件循环⇧

  • 执行至完成：每一个消息完整的执行后，其它消息才会被执行。
    • 优点：包括但不限于，当一个函数执行时，它不会被抢占，只有在它运行完毕之后才会去运行任何其它的代码，才能修改这个函数操作的数据。缺点在于当一个消息的处理事件太长时，会导致程序无法处理与用户的交互。良好的习惯是缩短每个消息处理时间，在可能的情况下尽量裁剪消息为最小单元。
  • 添加消息：在浏览器里，每当一个事件发生并且有一个事件监听器绑定在该事件上时，一个消息就会被添加进消息队列。如果没有事件监听器，这个事件将会消失。
  • 零延迟：零延迟并不意味着回调会立即执行。setTimeout()时间间隔参数设置为0并不是0毫秒后就立即调用回调函数。等待的时间取决于队列里待处理消息的数量。基本上，setTimeout需要等待当前队列中所有的消息都处理完毕之后才能执行，即使已经超出了时间间隔。
  • 多个运行时互相通信：一个web worker或者一个跨域的iframe都有自己的运行时，两个不同的运行时只能通过postMessage方法进行通信

  • 永不阻塞：JavaScript的事件循环模型与其它许多语言不同的有趣特性是，它永不阻塞。处理I/O通常通过事件和回调来执行，当等待一个缓存查询或一个xhr返回时，仍然可以处理其他事情，比如用户输入。

# 内存管理⇧

JavaScript是在创建变量（对象，字符串等）时自动进行了分配内存，并且在不断使用它们时“自动”释放，释放的过程称为垃圾回收。

# 内存生命周期⇧

不管什么程序语言，内存生命周期基本是一致的

  • 分配你需要的内存
  • 使用你分配的内存（读、写）
  • 不需要时将其释放归还

# JavaScript中的内存分配⇧

  • 值的初始化：JavaScript在定义变量时就完成了内存分配

  • 通过函数调用分配内存

    • 有些函数调用结果是分配对象内存：var date = new Date()
    • 有些方法是分配新变量或者新对象：var s = 'asdasd'; var s2 = s.substr(0,3); 或者 var a = arr1.concat(arr2)

  • 使用值：使用值的过程实际上是对分配的内存进行读取和写入操作。读取或写入可能是一个变量或者一个对象的属性值，甚至传递函数的参数。

# 垃圾回收⇧

当内存不在需要使用时释放，找到哪些分配的内存确实已经不再需要了，并释放出来。

  • 引用：在内存管理环境中，一个对象如果有访问另一个对象的权限(隐式或显式)就是一个对象引用另一个对象。
  • 引用计数垃圾收集：这是最初级别的垃圾收集算法。此算法把“对象是否不再需要”简化为“对象有没有被其它对象引用”。如果是零引用，对象就会被垃圾回收机制回收。
    • 循环引用：使用此算法有个限制，无法处理循环引用的事例。两个对象相互引用，调用之后没有用了本可以回收，此算法考虑它们互相有至少一次引用，于是不会回收。
    • IE6,7使用引用计数方式对DOM对象进行回收，常常造成对象被循环引用时内存发生泄漏。
  • 标记-清除算法：这个算法把“对象是否不再需要”简化为“对象是否可以获得”。这个算法比前一个好，因为零引用的对象总是不可获得的，但相反不一定。（现代浏览器中使用的方式是标记清除算法）
    • 解除了循环引用的限制
    • 新的限制：那些无法从根对象查询到的对象都将被清除。实践中很少碰到。
