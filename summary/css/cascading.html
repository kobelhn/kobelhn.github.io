    • 样式定义分为以下几种（权重依次递增）：
      • 选择器权重（权重依次递减）
        • !important 例外规则

          # css权重（优先级）

          举例情况是简单列举，实践过程中能更好的理解css权重(优先级)

          理解css权重(优先级)能更好的进行css模块化

          css模块化的重要作用就是让你的css层叠合理，维护和调整更加方便

          # 样式定义分为以下几种（权重依次递增）：

          • 浏览器用户自定义样式
          • 外联样式表：head内使用link引用的外部css文件
          • 内联样式表：head内使用style标签直接定义的css代码
          • 行内样式：直接在元素上使用style属性定义的css代码

          # 选择器权重（权重依次递减）

          • !important定义的样式
          • 行内样式
          • id选择器定义的样式
          • 类选择器定义的样式
          • 元素选择器定义的样式
          • 通配符选择器
          • 用户定义的浏览器样式
          • 浏览器默认样式

          # !important 例外规则

          • 不要滥用!important来解决样式覆盖问题，优先选择css规则优先级解决问题
          • 只有在覆盖全站或外部css的特定页面或场景时使用
          • 永远不要在你定义的插件中使用，永远不要在全站范围的css代码中使用

          # 层叠规则

          这里的样式定义理解为css属性 如 background width height 诸如此类

          使用过程中根据权重规则和书写顺序覆盖样式定义，样式的展示结果是多方规则层叠之后的结果

          • 针对同一个元素的相同样式定义，相同权重使用写在后面的样式定义，不同权重加载权重更高的样式定义
          • 针对同一个元素的不同样式定义，权重最高的或者写在后面的样式定义组合成为最终样式
