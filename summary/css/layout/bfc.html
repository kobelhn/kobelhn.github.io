        # BFC

        • 参考链接1 (opens new window)
        • 参考链接2 (opens new window)
        • 参考链接3 (opens new window)

        定义：块格式上下文(Block Formatting Context, BFC)是Web页面的可视化CSS渲染的一部分，是块盒子的布局过程发生的区域，也是浮动元素与其他元素交互的区域。

        # 块元素的流体特性

        块状水平元素，如div元素（下同），在默认情况下（非浮动、绝对定位等），水平方向会自动填满外部的容器；如果有margin-left/margin-right, padding-left/padding-right, border-left-width/border-right-width等，实际内容区域会响应变窄。

        # BFC原理（特性）

        • BFC内部元素在垂直方向上的边距会发生重叠
        • BFC不会与浮动元素重叠
        • BFC是独立的容器，不会影响内部元素，内部元素也不会影响外部元素
        • 计算BFC元素高度时，浮动元素也会参与计算

        # BFC产生条件

        • 根元素(<html>)
        • 浮动元素（元素的float不为none）
        • 绝对定位元素（元素的position为absolute或fixed）
        • 行内块元素（元素的display为inline-block）
        • overflow计算值(Computed)不为 visible 的块元素
        • 弹性元素（display为 flex 或 inline-flex元素的直接子元素）
        • 网格元素（display 为 grid 或 inline-grid 元素的直接子元素）
        • display 值为 flow-root 的元素
        • 表格单元格（元素的display为table-cell，HTML表格单元格默认为该值）
        • 表格标题（元素的display为table-caption，HTML表格标题默认为该值）
        • 匿名表格单元格元素（元素的 display 为 table、table-row、table-row-group、table-header-group、table-footer-group（分别是HTML table、row、tbody、thead、tfoot 的默认属性）或 inline-table）
        • contain 值为 layout、content或 paint 的元素
        • 多列容器（元素的 column-count 或 column-width 不为 auto，包括 column-count 为 1）
        • column-span 为 all 的元素始终会创建一个新的BFC，即使该元素没有包裹在一个多列容器中

        # margin穿透

        子元素想要距离父元素上边距有50px的距离

        <section class="sec">
  <article class="child">height: 50px;margin-top: 50px;</article>
</section>
<style>
.sec { background: green; }
.child { height: 50px;background: blue;margin-top: 50px; line-height: 50px;}
</style>

        解决方法：使父元素变为BFC，做如下代码改动就能够观察到子元素距离父元素的上边距，父元素的高度自适应为100px

        <style>
  .sec { background: green; overflow: hidden }
</style>

        # margin重合

        列表结构中期望两个item的上下间距为40px，设置每一个item设置margin-top:20px; margin-bottom: 20px;

        <section class="margin">
  <p>测试测试测试</p>
  <p>测试测试测试</p>
</section>
<style>
.margin {background: #ebf6fd;}
.margin p { margin: 20px auto;background: #09d; line-height: 40px;}
</style>

    <p>测试测试测试</p>

        # 两栏自适应布局

        float的包裹性会使得右边的文字环绕左边的图片显示，给右边的元素添加BFC特性，即可使得右边的元素与浮动图片隔离开来，且BFC元素依旧保留块元素的流体特性，利用这个原理可以方便实现两栏自适应布局

        <section class="layout">
  <img src="img/bqb1.jpg" class="left" alt="" />
  <!-- <div class="right"> -->
  <div class="right" style="overflow: hidden">
    傻人有傻福
  </div>
</section>
<style>
.layout {background: #ebf6fd;margin-top: 20px;}
.layout .left { float: left; width: 150px;height: 168px;}
.layout .right {background: #09d;text-align: left; }
</style>

        # 清除浮动

        父元素包含浮动元素且高度不固定时，出现浮动溢出问题

        解决方法：使父元素变为BFC，添加清除浮动的类

        .clearfix {*zoom: 1 }
.clearfix:after {content: '';display: table; clear: both }
