      # this、原型、原型链

      # 对象原型/原型链

      • __proto__：JavaScript中的对象都有一个特殊的__proto__内置属性，其实就是对于其他对象的引用。几乎所有的对象在创建时__proto__属性都会被赋予一个非空的值。所有普通的__proto__链最终都会指向内值的Object.prototype。
      • 引用对象的属性第一步是检查对象本身是否有这个属性，如果有就使用它，如果无法在对象本身找到需要的属性就会继续访问对象的原型链，这个过程会持续到找到匹配的属性或者查找完整的原型链。通过各种语法进行属性查找时都会查找原型链，直到找到属性或者查找完整原型链
      • 属性屏蔽（其他语言里的重写）例：obj.a = a
        • 如果原型链上层存在同名属性，如果这个同名属性只是普通数据访问属性，没有被标记为只读，那就会在当前对象中添加一个新属性
        • 如果原型链上层存在同名属性，这个同名属性又被设置为只读，那这个属性不会被添加，也不会改变上层的同名属性，这条语句相当于被忽略，严格模式下程序会报错。
        • 如果原型链上层存在同名属性，这个同名属性是一个setter，那么会执行setter，属性不会添加到当前对象，也不会改变setter
      • 继承
        • 原型继承：当前对象可以访问和引用它原型链上的属性
        • 构造函数继承：所有带new关键字的函数调用就是构造函数
      • 性能：
        • 在原型链上查找属性比较耗时，对性能有副作用，这在性能要求苛刻的情况下很重要。在试图访问不存在的属性时会遍历整个原型链
        • hasOwnProperty()是JavaScript中唯一一个处理属性并且不会遍历原型链的方法。另一种这样的方法是：Object.keys() 只会遍历当前对象的属性而不会遍历原型链。

      # this全面解析

      • this是在运行时绑定的，并不是在编写时绑定，它的上下文取决于函数调用时的各种条件。this的绑定和函数声明的位置没有任何关系，只取决于函数的调用方式。函数调用时，会创建一个执行上下文，这个上下文会包含函数在哪里被调用，函数的调用方法，函数的参数等信息，this就是执行上下文中的一个属性，会在函数执行的过程中用到。

      • 绑定方式：

        • 默认绑定：独立函数调用。严格模式下，全局对象无法使用默认绑定，程序会抛错误。

        • 隐式绑定：调用位置是否有上下文对象，或者说是否被某个对象拥有或者包含。

          function foo() { console.log(this.a) }; const obj = { foo, a: 2 }; obj.foo(); // 2

        • 显式绑定：call() apply()方法的参数，第一个参数是一个对象，会把这个对象绑定到this，在调用时指定这个this

          • 硬绑定：使用Function.prototype.bind()方法，直接绑定函数调用时的this

        • new绑定：使用new来调用函数（构造函数调用）时

          • 创建（构造）一个全新的对象
          • 这个新对象会被执行原型连接
          • 这个对象会被绑定到函数调用的this
          • 如果函数没有返回其他对象，那么new表达式中的函数调用，会自动返回这个新对象

      • 绑定优先级（优先级依次降低）

        • new绑定
        • 显式绑定，硬绑定
        • 隐式绑定
        • 默认绑定

      • 绑定例外：

        • 如果把null或者undefined作为this的绑定对象传入call,apply或bind中，这些值在调用时会被忽略而应用默认绑定规则

          function foo() { console.log(this.a) }; var a = 2; foo.call(null); // 2

        • 有可能会在有意无意间创建一个函数的间接引用，这时也会应用默认绑定规则

          function foo() { console.log(this.a) };
var a = 2;
var o = { a: 3, foo: foo };
var p = { a: 4 };
o.foo(); // 3
(p.foo = o.foo)(); // 2

      • 箭头函数：箭头函数不使用this的四种绑定规则，而是根据外层（函数或全局）作用域来决定this。
